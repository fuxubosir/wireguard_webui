# WireGuard WebUI v1.12.63

这是一个面向现场部署的 WireGuard Web 管理工具。

当前发布包采用“根目录极简 + bundle 核心目录”的结构：

```text
wg-webui-v1.12.63/
├── wg-webui.sh      # 唯一用户入口，直接执行后打开交互菜单
├── README.md        # 使用说明
├── CHANGELOG.md     # 更新记录
├── VERSION          # 版本号
├── manifest.json    # 发布包清单
└── bundle/          # WebUI 核心程序、内部脚本、工具、文档、测试
```

## 快速使用

解压后进入目录，直接执行：

```bash
sudo bash wg-webui.sh
```

之后按菜单选择：

```text
1) 安装 / 首次部署
2) 升级 WebUI
3) 运行诊断
4) 卸载 / 清理
5) 查看 WebUI 服务状态
6) 重启 WebUI 服务
7) 查看 WebUI 日志
0) 退出
```

日常使用不需要记 `install.sh`、`upgrade.sh`、`doctor.sh`、`uninstall.sh` 这些脚本名，也不需要进入 `bundle/`。

## 目录说明

`bundle/` 是核心目录，里面包含：

```text
bundle/app/        WebUI 后端、页面模板、静态资源和核心逻辑
bundle/config/     默认配置样例
bundle/scripts/    安装、升级、诊断、卸载的真实执行脚本
bundle/tools/      修复、同步、清理、发布检查和打包工具
bundle/docs/       开发和路线文档
bundle/tests/      基础测试
```

根目录只保留一个入口 `wg-webui.sh`，这样现场部署时更清爽，也避免用户不知道该执行哪个脚本。

## 安装

```bash
tar -xzf wg-webui-v1.12.63.tar.gz
cd wg-webui-v1.12.63
sudo bash wg-webui.sh
```

选择：

```text
1) 安装 / 首次部署
```

安装过程会自动处理 WebUI 程序目录、配置目录、Python 依赖、systemd 服务等内容。

## 升级

进入新版发布包目录：

```bash
sudo bash wg-webui.sh
```

选择：

```text
2) 升级 WebUI
```

然后输入升级包路径，例如：

```text
/tmp/wg-webui-v1.12.63.tar.gz
```

升级原则：

- 只升级 WebUI 程序
- 保留 `/etc/wg-webui/config.json`
- 保留数据库和现有配置
- 不随意重启 WireGuard 隧道
- 失败时尽量回滚

## 诊断

```bash
sudo bash wg-webui.sh
```

选择：

```text
3) 运行诊断
```

用于检查 WebUI 服务、WireGuard 相关配置、NAT/FORWARD、系统环境等。

## 卸载

```bash
sudo bash wg-webui.sh
```

选择：

```text
4) 卸载 / 清理
```

卸载脚本内部会继续区分不同清理方式，避免误删生产 WireGuard 配置。

## 服务维护

仍然通过同一个入口：

```bash
sudo bash wg-webui.sh
```

可以在菜单里查看服务状态、重启 WebUI 服务、查看最近日志。


## v1.12.63 页面细节优化

- 右上角提示框下移，避免新增站点、权限变更等操作后遮挡顶部“配置应用”按钮。
- 系统部署包区域改为两个说明卡片：完整部署包和一键卸载脚本，每个说明框下方对应自己的下载按钮。
- 日志清理区域改为两个并列操作卡片：扫描按钮对应“可清理项目”，清理按钮对应“可释放空间”，删除底部多余提示小字。

## GitHub 与后续 Vue 重构

这个版本的重点是整理发布包结构：

- 根目录只保留唯一入口
- 核心程序继续放在 `bundle/`
- 保留开发文档、测试和打包检查工具
- 为后续 Vue3 + Element Plus 大版本重构做准备

后续 Vue 版本仍建议保持现场使用方式不变：一个压缩包、一个入口脚本、一个 Web 页面。


## v1.12.63 页面与安全配置优化

- 系统管理里的维护策略和安全账号改成“展示更多”样式，日常配置默认更清爽。
- 安全配置默认只展示常用项：登录失败次数、会话有效期（分钟）；失败统计时间和锁定时长收进“展示更多安全参数”，一般保持默认即可。
- 会话有效期默认改为 30 分钟，到期后自动返回登录界面。
- WebUI 在线升级启动后，升级完成会引导返回登录界面，避免旧会话继续停留在旧页面。
- 系统部署包和日志清理区域统一卡片高度和按钮对齐；页面表头和下方内容间距按仪表盘风格统一。


## v1.12.63 页面调整

- 系统管理里的“维护策略”和“安全账号”现在由一个居中的“展示更多”按钮统一控制，点开后两个配置区一起显示。
- “系统部署包”和“日志清理”使用一致的双卡片布局，高度和按钮位置保持对齐。
- 高级设置里增加“重启 WebUI”按钮，只重启 WebUI 管理服务，不重启 WireGuard，不影响现有 VPN 隧道。

## v1.12.63 配置中心调整

- 根目录仍只保留 `wg-webui.sh` 统一入口。
- 配置中心右上角按钮统一命名为“保存配置”。
- 安全配置恢复完整展示，登录失败次数、统计窗口、锁定时长、会话有效期都直接可见。
- “展示更多”统一显示维护策略、安全账号和高级设置。
- 修改账号或密码后，填写当前密码、新密码和确认密码，再点击“保存配置”；保存成功后会回到登录页。
- 高级设置默认只读，只有点击“解锁配置修改”后才显示“重启 WebUI”按钮。重启 WebUI 只重启管理后台，不会重启 WireGuard。

## v1.12.63 页面间距修复

- 六个主页面顶部内容间距统一，以仪表盘为参照。
- 移除页面切换区之间遗留的固定空白，系统管理页不会再在标题下方出现大块空白。

## 统一入口脚本

根目录只保留 `wg-webui.sh` 作为日常入口。解压后执行：

```bash
sudo bash wg-webui.sh
```

脚本会进入交互菜单，可直接选择安装、升级、诊断、查看状态、重启 WebUI、查看日志和卸载清理。每个一级菜单都有退出入口，执行完成后会返回菜单。

站点部署包同样推荐执行：

```bash
sudo bash site.sh
```

站点脚本会提供部署、查看状态、卸载清理等选项，并保留返回/退出入口。


## v1.12.63 站点部署包结构调整

站点部署包现在和系统部署包保持一致：根目录只保留 `site.sh`、说明文件和清单文件，真正的安装、卸载、模板和兼容工具都放在 `bundle/` 内部。现场使用时只需要执行：

```bash
sudo bash site.sh
```

菜单内提供部署、状态查看、卸载/清理和日志查看，不再需要在根目录单独执行 `install.sh` 或 `uninstall.sh`。

## v1.12.63 权限管理搜索框调整

- 权限管理搜索框样式与用户管理保持一致。
- 搜索框左移，贴近“权限管理”标题，右侧批量授权、站点授权和同步按钮保持右对齐。


## v1.12.63 运维工具页面

运行日志页已调整为“运维工具”，提供一键诊断、服务状态、WireGuard 状态、路由/转发检查、快捷指令和必要日志入口。

## v1.12.63 运维工具调整

- 运维工具顶部按钮现在会显示当前选中状态，切换工具时底色同步变化。
- “WireGuard 状态”继续用于查看接口、Peer、握手和流量；“连接快照”移到必要日志区域，避免顶部功能重复。
- 新增 Ping 检测，可直接输入 IP 或域名测试连通性，适合快速检查用户节点、站点节点、现场网关或内网设备。
- Ping 工具只执行固定 ping 命令，不开放自定义命令入口。

## v1.12.63 站点网段总览

站点管理页面新增“站点网段总览”板块，用卡片集中展示每个站点的 VPN 地址、在线状态和内网网段。该板块会跟随站点搜索同步过滤，适合站点和网段数量变多后的日常查看与排查。每个站点卡片可直接编辑网段或下载部署包。

## v1.12.63 列表 / 卡片显示

用户列表、站点列表、权限管理支持在“列表”和“卡片”两种显示方式之间切换。站点网段仍在站点列表中以 `AllowedIPs = ...` 形式展示，不再额外占用一个单独总览板块。
## v1.12.63 列表 / 卡片显示优化

- 站点列表网段显示恢复为原有简洁方式，不再强行加 `AllowedIPs =` 前缀。

- 卡片里只保留必要信息，避免显示过多字段导致页面拥挤。
