# 配置说明

主配置文件位于：

```text
/etc/wg-webui/config.json
```

## 常用配置

- `server_endpoint`：客户端和站点连接服务端时使用的公网 IP/域名和端口。
- `client_dns`：写入用户客户端配置的 DNS，可为空。
- `reserved_client_allowed_ips`：固定下发给用户客户端的网段，例如公司内网、保留网段。
- `client_allowed_ips`：旧版本兼容字段。配置中心会把它和 `reserved_client_allowed_ips` 合并展示为“固定下发网段”，保存后统一写回 `reserved_client_allowed_ips`。
- `online_threshold_seconds`：在线状态判定时间。

这些配置在“系统管理 -> 配置中心”中可直接修改并保存。安全配置保存后会即时更新运行时参数，不需要重启 WebUI。

## 高级设置

以下配置会影响 WireGuard 运行、客户端连接或地址分配，默认折叠且只读：

- `wg_if`：WebUI 管理的 WireGuard 服务端接口。
- `wg_cidr` / `wg_net`：WireGuard VPN 地址池。
- `server_endpoint`：服务端连接地址。
- `site_ip_start` / `site_ip_end`：站点地址池范围。
- `user_ip_start` / `user_ip_end`：用户地址池范围。

需要修改时，先展开高级设置，点击“解锁配置修改”，确认风险后再保存。保存后按页面提示点击“应用配置”或重启相关服务。

## 展示元数据

- `user_owners`：用户归属人。
- `site_remarks`：站点备注。
- `user_site_permissions`：用户访问站点权限。

这些是 WebUI 展示和授权数据，保持在 `config.json` 中，不写入 WireGuard 原生配置，避免污染现场配置。

## 账号安全

账号安全模块支持在 WebUI 中修改用户名和密码。新密码会以 PBKDF2-SHA256 哈希写入 `webui_password_hash`。新安装不再默认用 systemd 环境变量接管账号；只有管理员主动设置自定义账号环境变量时，WebUI 才会提示账号由服务环境变量管理。

默认密码提醒只在登录页点到为止，不阻止登录，不在每个页面重复显示。`cookie_secure` 仍作为配置文件兼容项保留，适合 HTTPS 反向代理环境手动配置，普通页面不再显示这个专业选项。


## v2.0.0 页面说明

系统管理页中，“系统部署包”区域现在使用两个说明卡片分别对应完整部署包和一键卸载脚本下载；“日志清理”区域中，扫描和清理按钮分别对齐对应的统计卡片。右上角操作提示会下移显示，避免遮挡顶部“配置应用”按钮。

## v2.0.0 配置中心调整

- 根目录仍只保留 `wg-webui.sh` 统一入口。
- 配置中心右上角按钮统一命名为“保存配置”。
- 安全配置恢复完整展示，登录失败次数、统计窗口、锁定时长、会话有效期都直接可见。
- “展示更多”统一显示维护策略、安全账号和高级设置。
- 修改账号或密码后，填写当前密码、新密码和确认密码，再点击“保存配置”；保存成功后会回到登录页。
- 高级设置默认只读，点击“解锁配置修改”后才显示底部操作按钮；保存高级设置后会保持解锁，可继续修改或点击“重启 WebUI”。重启 WebUI 只重启管理后台，不会重启 WireGuard。


## v2.0.0 运维工具页

运行日志页调整为运维工具页，集中提供一键诊断、服务状态、WireGuard 状态、路由/转发检查、快捷指令和必要日志。

## v2.0.0 运维工具调整

- 运维工具顶部按钮现在会显示当前选中状态，切换工具时底色同步变化。
- “WireGuard 状态”继续用于查看接口、Peer、握手和流量；“连接快照”移到必要日志区域，避免顶部功能重复。
- 新增 Ping 检测，可直接输入 IP 或域名测试连通性，适合快速检查用户节点、站点节点、现场网关或内网设备。
- Ping 工具只执行固定 ping 命令，不开放自定义命令入口。
