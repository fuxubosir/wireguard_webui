# 安装与维护

发布包根目录只有一个用户入口：

```bash
sudo bash wg-webui.sh
```

执行后会进入交互菜单：

```text
1) 安装 / 首次部署
2) 升级 WebUI
3) 运行诊断
4) 卸载 / 清理
5) 查看 WebUI 服务状态
6) 重启 WebUI 服务
7) 查看 WebUI 日志
0) 退出
```

现场部署、升级、诊断、卸载都从这个菜单进入。不要直接执行 `bundle/scripts/` 下的内部脚本，除非是在排查问题或开发调试。

## 首次安装

```bash
tar -xzf wg-webui-v1.12.62.tar.gz
cd wg-webui-v1.12.62
sudo bash wg-webui.sh
```

选择 `1) 安装 / 首次部署`。

## 升级

解压新版包后执行：

```bash
sudo bash wg-webui.sh
```

选择 `2) 升级 WebUI`，再输入升级包路径。

## 诊断

执行菜单后选择 `3) 运行诊断`。

## 卸载

执行菜单后选择 `4) 卸载 / 清理`。

卸载脚本内部会继续做确认和模式区分，避免误删 WireGuard 生产配置。


## v1.12.62 页面说明

系统管理页中，“系统部署包”区域现在使用两个说明卡片分别对应完整部署包和一键卸载脚本下载；“日志清理”区域中，扫描和清理按钮分别对齐对应的统计卡片。右上角操作提示会下移显示，避免遮挡顶部“配置应用”按钮。

## v1.12.62 站点部署包结构调整

站点部署包现在和系统部署包保持一致：根目录只保留 `site.sh`、说明文件和清单文件，真正的安装、卸载、模板和兼容工具都放在 `bundle/` 内部。现场使用时只需要执行：

```bash
sudo bash site.sh
```

菜单内提供部署、状态查看、卸载/清理和日志查看，不再需要在根目录单独执行 `install.sh` 或 `uninstall.sh`。


## v1.12.62 运维工具页

运行日志页调整为运维工具页，集中提供一键诊断、服务状态、WireGuard 状态、路由/转发检查、快捷指令和必要日志。
