# 后续路线

当前版本已经适合作为稳定基线。后续迭代建议保持“小步、可回退、先测试”的节奏。

## 优先级高

- 登录安全：保留默认密码提醒，但只在登录页简洁提醒，不阻止内网使用。
- 配置安全：继续将影响运行的配置放入高级设置，默认只读，解锁后修改。
- 部署安全：安装/卸载脚本继续只清理自己创建的服务、接口、配置和规则。
- 升级安全：保持在线升级预检、备份、失败回滚和不重启 WireGuard 隧道。

## 可持续优化

- 运行日志增加更友好的连接变化摘要。
- 增加配置导入/导出预检，避免地址池和权限误配置。
- 增加用户备注或标签。
- 为手机端继续做真实设备截图回归。

## 暂不建议

- 不把专业路径、监听地址等危险配置暴露给普通日常操作。
- 不把站点备注、用户归属人等展示数据写进 WireGuard 原生配置。
- 不在登录后反复弹默认密码提醒。
