# 运维操作

本文面向日常维护和问题排查。

## 日常检查

建议重点关注：

- WebUI 服务是否运行。
- WireGuard 接口是否运行。
- 用户和站点是否有最近握手时间。
- 地址池是否还有可用地址。
- 页面顶部是否存在待应用配置提示。

## 新增用户

1. 在 WebUI 中进入用户管理。
2. 新增用户并生成配置。
3. 下载客户端配置或二维码。
4. 如需访问站点，在权限管理中授予对应站点权限。
5. 权限变更后同步 AllowedIPs 并应用配置。

## 新增站点

1. 在站点管理中新增站点。
2. 填写站点名称、备注和需要暴露的内网网段。
3. 下载站点部署包。
4. 在现场服务器执行站点部署脚本。
5. 回到 WebUI 应用配置并检查连通性。

## 权限管理

用户访问站点的范围通过权限管理维护。调整权限后建议按顺序执行：

1. 保存授权。
2. 同步用户网段。
3. 应用配置。
4. 测试用户侧是否能访问目标站点网段。

## 运维工具

WebUI 内置运维工具用于快速排查：

- 服务状态：检查 WebUI 和 WireGuard systemd 状态。
- WireGuard 状态：查看接口、Peer、握手和流量。
- 路由/转发/NAT：检查站点和用户访问链路。
- 日志：查看 WebUI 和 WireGuard 最近日志。
- Ping / TCP 检测：测试单个 IP、域名或端口连通性。
- 网段扫描：对小网段做快速可达性检查。

## 常见问题

### 新增站点后不通

先确认已经点击“应用配置”，再检查站点部署是否成功、站点内网网段是否填写正确、现场网卡和路由是否正常。

### 用户不能访问站点

检查该用户是否已授予站点权限，是否执行了 AllowedIPs 同步，客户端是否重新导入或刷新了配置。

### 页面样式异常

强制刷新浏览器缓存，确认加载的是当前版本静态资源，例如：

```text
app.css?v=2.0.0
app.js?v=2.0.0
```

### WebUI 无法访问

检查 WebUI 服务状态和日志：

```bash
sudo bash wg-webui.sh
```

选择服务状态或日志菜单项。
