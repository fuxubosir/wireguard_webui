# 运维操作

## 日常检查

- 仪表盘 WireGuard 服务为运行中。
- 在线用户、在线站点有底色高亮，最近握手时间正常。
- 顶部“应用配置”不是长期 pending。
- 地址池剩余数量充足。

## 新增用户

1. 在“用户管理”新增用户。
2. 查看二维码或配置文件。
3. 按需在“权限管理”调整访问范围。
4. 修改权限后按权限同步用户网段，再点击“应用配置”。

## 新增站点

1. 在“站点管理”新增站点，可先只填站点名称和备注。
2. 下载站点部署包。
3. 现场执行 `sudo ./site.sh`。
4. 回到 WebUI 点击顶部“应用配置”，再测试连通性。

站点内网网段和现场网卡不是必填项。部署包安装时会自动识别现场网卡并添加 NAT/转发；需要让用户访问站点内网时，再补充站点网段并应用配置。

## 权限管理

权限页面显示站点备注，未配置备注时显示站点名称。全部授权显示为“全部站点”，避免站点增多后被大量 IP 网段淹没。

修改访问范围后建议按顺序操作：

1. 保存授权。
2. 按权限同步用户网段。
3. 点击顶部“应用配置”。

## 日志与排错

- “连接快照”用于看最近握手、端点和流量。
- “WireGuard 服务”用于看 `wg-quick@接口` 的 systemd 日志。
- “WebUI 服务”用于看 WebUI 后端日志。

常见问题：

- 新增站点后不通：先确认已经点击“应用配置”。
- UI 样式异常：强制刷新浏览器缓存，确认加载的是 `app.css?v=2.0.0`。
- 站点包安装失败：查看 `.site-install-state` 和 `journalctl -u wg-quick@实际接口名`。
- 服务器同时做服务端和站点接入端：使用不同 WireGuard 接口，避免复用 `wg0`。


## v2.0.0 页面说明

系统管理页中，“系统部署包”区域现在使用两个说明卡片分别对应完整部署包和一键卸载脚本下载；“日志清理”区域中，扫描和清理按钮分别对齐对应的统计卡片。右上角操作提示会下移显示，避免遮挡顶部“配置应用”按钮。

## v2.0.0 配置中心调整

- 根目录仍只保留 `wg-webui.sh` 统一入口。
- 配置中心右上角按钮统一命名为“保存配置”。
- 安全配置恢复完整展示，登录失败次数、统计窗口、锁定时长、会话有效期都直接可见。
- “展示更多”统一显示维护策略、安全账号和高级设置。
- 修改账号或密码后，填写当前密码、新密码和确认密码，再点击“保存配置”；保存成功后会回到登录页。
- 高级设置默认只读，点击“解锁配置修改”后才显示底部操作按钮；保存高级设置后会保持解锁，可继续修改或点击“重启 WebUI”。重启 WebUI 只重启管理后台，不会重启 WireGuard。

## v2.0.0 站点部署包结构调整

站点部署包现在和系统部署包保持一致：根目录只保留 `site.sh`、说明文件和清单文件，真正的安装、卸载、模板和兼容工具都放在 `bundle/` 内部。现场使用时只需要执行：

```bash
sudo bash site.sh
```

菜单内提供部署、状态查看、卸载/清理和日志查看，不再需要在根目录单独执行 `install.sh` 或 `uninstall.sh`。


## v2.0.0 运维工具页

运行日志页调整为运维工具页，集中提供一键诊断、服务状态、WireGuard 状态、路由/转发检查、快捷指令和必要日志。

## v2.0.0 运维工具调整

- 运维工具顶部按钮现在会显示当前选中状态，切换工具时底色同步变化。
- “WireGuard 状态”继续用于查看接口、Peer、握手和流量；“连接快照”移到必要日志区域，避免顶部功能重复。
- 新增 Ping 检测，可直接输入 IP 或域名测试连通性，适合快速检查用户节点、站点节点、现场网关或内网设备。
- Ping 工具只执行固定 ping 命令，不开放自定义命令入口。
